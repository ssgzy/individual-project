What I Learned from Training a Sokoban Agent

2026-03-05 · Reinforcement Learning

What looked like a standard RL assignment became a practical lesson in disciplined experimentation. Sokoban has sparse rewards and deceptive solutions, so early overfitting is easy and hard to notice.

The first lesson was baseline control. I kept a strict baseline chain, then changed one variable at a time: reward shaping, replay strategy, learning rate, and evaluation policy. This avoided confounding results, which is often the hidden failure point in student-level RL projects.

The second lesson was budget discipline. RL quickly burns compute if uncontrolled. I added early termination metrics and per-run checkpoints, then prioritized experiments that improved learning stability rather than only peak score.

The final improvement came from reproducibility habits: deterministic seeds, fixed splits, and a clear result note format. These do not always raise absolute performance immediately, but they make iteration faster and less confusing later.
